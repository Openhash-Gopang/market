# 고팡(Gopang) 업체 등록 매뉴얼 및 표준
**버전** v1.0  
**작성일** 2026-06-12  
**작성자** AI City Inc. 팀 주피터  
**적용 범위** Gopang K-Market 및 전체 하위 시스템 업체 등록

---

## 1. 개요

고팡 플랫폼에 업체를 등록하려면 `user_profiles` 테이블에 표준화된 형식으로 데이터를 입력해야 한다. 이 문서는 한국표준산업분류(KSIC, 제10차 개정, 통계청 고시 2017-13호)와 국세청 업종코드 체계를 기준으로 업체 등록 및 검색의 표준 프로세스를 정의한다.

---

## 2. 핵심 개념 정리

### 2.1 한국표준산업분류(KSIC) 코드 체계

통계청이 고시하는 KSIC는 5단계 계층 구조로 이루어진다.

| 단계 | 명칭 | 자릿수 | 예시 |
|------|------|--------|------|
| 1단계 | 대분류 | 알파벳 1자리 | I — 숙박 및 음식점업 |
| 2단계 | 중분류 | 숫자 2자리 | 56 — 음식점 및 주점업 |
| 3단계 | 소분류 | 숫자 3자리 | 561 — 음식점업 |
| 4단계 | 세분류 | 숫자 4자리 | 5612 — 중식 음식점업 |
| 5단계 | 세세분류 | 숫자 5자리 | 56120 — 중식 음식점업 |

### 2.2 국세청 업종코드

국세청 업종코드는 KSIC를 기반으로 세금 징수·경비율 관리를 위해 부여하는 **6자리 숫자**다. 사업자등록증에는 기재되지 않으며 홈택스에서 조회 가능하다.

### 2.3 업태 vs 종목 vs 업종

| 용어 | 정의 | KSIC 대응 | 예시 |
|------|------|-----------|------|
| 업태 | 판매·영업 방식의 대분류 | 대분류(알파벳) | 숙박 및 음식점업 |
| 종목 | 업태 내 세부 품목·서비스 | 세분류~세세분류 | 중식 음식점업 |
| 업종코드 | 업태+종목을 통합한 국세청 6자리 코드 | — | 552101 |

### 2.4 entity_type 정의 (고팡 내부)

`user_profiles.entity_type` 컬럼은 고팡 플랫폼 내 주체 유형을 구분한다. **KSIC와 직접 매핑되지 않으며**, 고팡 내부 분류다.

| entity_type 값 | 의미 | 해당 주체 |
|----------------|------|-----------|
| `individual` | 개인 사용자 | 일반 소비자, 개인 판매자 |
| `business` | 사업자 | 개인사업자, 법인사업자 |
| `org` | 비영리·공공기관 | 관공서, 협회, 학교 등 |
| `platform` | 플랫폼 내부 주체 | gopang-platform 등 |

> ⚠️ **오늘의 교훈**: `search_entities` RPC의 `p_entity_type` 파라미터에 `'org'`를 하드코딩하면 `'business'` 업체가 검색에서 누락된다. 반드시 `null`로 설정하거나 명시적으로 지정해야 한다.

## 3. user_profiles 테이블 컬럼 표준

| 컬럼 | 타입 | 필수 | 표준값 기준 |
|------|------|------|------------|
| `primary_guid` | text | ✅ | Gopang IPv6 GUID 또는 테스트용 더미 ID |
| `guid` | text | ✅ | primary_guid와 동일 (단일 기기 기준) |
| `handle` | text | ✅ | `@영문소문자숫자` 형식. 예: `@geumneung` |
| `name` | text | ✅ | 사업자등록증 상호명 |
| `entity_type` | text | ✅ | `individual` / `business` / `org` / `platform` |
| `occupation` | text | ✅ | KSIC 세분류 명칭. 예: `중식 음식점업` |
| `services` | text[] | ✅ | 주요 메뉴·서비스 키워드 배열 (검색 인덱스) |
| `address` | text | ✅ | 도로명주소 전체. 예: `제주특별자치도 제주시 한림읍 금능리 123` |
| `phone` | text | 권장 | 국내 표준: `064-123-4567` |
| `website` | text | 권장 | `https://users.gopang.net/profile.html?handle=@핸들` |
| `is_public` | boolean | ✅ | 검색 노출 여부. 등록 시 `true` |
| `l1_node` | text | ✅ | 관할 L1 노드 ID. 한림읍: `KR-JEJU-JEJU-HANLIM` |
| `extra` | jsonb | 권장 | 메뉴, 영업시간, 추가정보 (하단 표준 스키마 참조) |
| `registered_l1` | boolean | — | L1 노드 등록 완료 시 `true` |

## 5. services 필드 표준

`services` 필드는 `occupation`보다 세부적인 **검색 키워드 배열**이다. `search_entities` RPC의 키워드 검색(`p_keyword`)이 이 배열을 참조한다.

**작성 규칙**:
- 주요 메뉴·상품명을 개별 항목으로 기재
- 동의어·유사어 포함 권장
- 최대 20개 이내

```sql
-- 예시: 중식당 금능반점
services = ARRAY['짜장면', '짜장밥', '짬뽕', '탕수육', '볶음밥', '중식', '중화요리']
```

## 7. 업체 등록 표준 프로세스

### 7.1 등록 전 준비 사항

1. **사업자등록증 확인** — 상호명, 사업자등록번호, 업태, 종목, 사업장 주소 확인
2. **KSIC 코드 확인** — 통계청 분류포털(kssc.kostat.go.kr)에서 업종에 맞는 세분류 코드 조회
3. **국세청 업종코드 확인** — 홈택스(hometax.go.kr)에서 6자리 업종코드 확인
4. **handle 결정** — `@영문소문자숫자` 형식, 중복 없이 결정
5. **메뉴 및 서비스 목록 준비** — K-Market 주문 연동 시 필수

### 7.2 Supabase SQL 등록 절차

**Step 1 — 중복 확인**
```sql
SELECT primary_guid, handle, name
FROM user_profiles
WHERE handle = '@핸들명' OR name = '상호명';
```

**Step 2 — users 테이블 선행 확인**
```sql
-- user_profiles보다 users 테이블이 먼저 존재해야 함 (search_entities JOIN 조건)
SELECT guid FROM users WHERE guid = '등록할_GUID';
```

**Step 3 — user_profiles INSERT**
```sql
INSERT INTO user_profiles (
  primary_guid,
  guid,
  handle,
  name,
  entity_type,
  occupation,
  services,
  address,
  phone,
  website,
  is_public,
  l1_node,
  extra
) VALUES (
  'pguid-업체식별자',
  'pguid-업체식별자',          -- guid = primary_guid (단일 기기 기준)
  '@핸들명',
  '상호명',
  'business',                  -- 사업자는 반드시 'business'
  '중식 음식점업',              -- KSIC 세분류 명칭
  ARRAY['짜장면', '짬뽕', '탕수육'],   -- 검색 키워드
  '제주특별자치도 제주시 한림읍 도로명주소',
  '064-000-0000',
  'https://users.gopang.net/profile.html?handle=@핸들명',
  true,
  'KR-JEJU-JEJU-HANLIM',
  '{
    "menu": [
      {"id": "m001", "name": "짜장면", "price": 6000, "available": true}
    ],
    "business_registration_number": "123-45-67890",
    "business_type": "개인사업자",
    "tax_type": "일반과세자",
    "ksic_code": "56120",
    "nts_industry_code": "552201",
    "gdc_accepted": true
  }'::jsonb
);
```

**Step 4 — 등록 검증**
```sql
SELECT * FROM search_entities(
  p_keyword    => '짜장면',
  p_occupation => '중식 음식점업',
  p_address    => '한림읍'
);
-- 등록한 업체가 결과에 포함되어야 함
```

### 7.3 등록 후 체크리스트

- [ ] `search_entities` 결과에 업체 포함 확인
- [ ] `users.gopang.net/profile.html?handle=@핸들명` 접속 시 프로필 정상 표시
- [ ] 메뉴 목록 렌더링 확인 (K-Market 주문 연동 시)
- [ ] `gdc_accepted = true` 확인 (GDC 결제 업체)
- [ ] `l1_node` 값이 관할 노드와 일치 확인

## 8. search_entities RPC 호출 표준

### 8.1 파라미터 전체 목록

```javascript
// market/webapp.html — 올바른 호출 방식
const rpcRes = await fetch(`${PROXY_BASE}/search`, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    p_keyword:      '짜장면',           // 메뉴·상호 키워드
    p_occupation:   '중식 음식점업',    // KSIC 세분류 명칭 (부분 일치)
    p_address:      '한림읍',           // 주소 (부분 일치)
    p_entity_type:  null,               // ⚠️ null 권장 — 'org' 하드코딩 금지
    p_limit:        5,
    p_offset:       0,
    p_lang_code:    'ko',
    p_sort:         'rating',           // rating | distance | review_count
    p_lat:          33.3863,            // GPS 좌표 (거리 정렬 시)
    p_lng:          126.2327,
    p_gdc_only:     null,               // true = GDC 결제 업체만
    p_trust_min:    null,               // 'L0'~'L3' 최소 신뢰 등급
    p_exclude_guid: '현재_사용자_guid',
    p_l1_node:      null,               // 'KR-JEJU-JEJU-HANLIM' 등
    p_handle:       null,
    p_nickname:     null,
    p_primary_guid: null,
  }),
});
```

### 8.2 p_entity_type 사용 지침

| 상황 | 권장값 |
|------|--------|
| 일반 업체 검색 (기본) | `null` — 모든 entity_type 검색 |
| 음식점·소매점 등 사업자만 | `'business'` |
| 관공서·공공기관만 | `'org'` |
| 개인 판매자만 | `'individual'` |

> ⚠️ `'org'`를 기본값으로 사용하면 `entity_type = 'business'`인 대부분의 상업 업체가 누락된다.

## 9. L1 노드 코드 체계

업체 등록 시 `l1_node` 필드는 업체 소재지 기준 L1 노드 ID를 기재한다.

| 지역 | l1_node 값 |
|------|-----------|
| 제주 한림읍 (파일럿) | `KR-JEJU-JEJU-HANLIM` |
| (향후 확장) 제주시 | `KR-JEJU-JEJU-JEJU` |
| (향후 확장) 서귀포시 | `KR-JEJU-SEOGWIPO` |

**형식**: `KR-{광역시도코드}-{시군구코드}-{읍면동코드}`

## 10. 테스트 데이터 표준 (파일럿: 한림읍)

파일럿 테스트 업체는 아래 naming convention을 따른다.

| 필드 | 규칙 | 예시 |
|------|------|------|
| `primary_guid` | `dummy-hanlim-{3자리 순번}` | `dummy-hanlim-003` |
| `handle` | `@{영문지역명}{업종약어}` | `@geumneung` |
| `name` | 실제 또는 가상 상호명 | `금능반점` |
| `l1_node` | `KR-JEJU-JEJU-HANLIM` 고정 | — |

## 11. 자주 발생하는 오류 및 해결

| 오류 | 원인 | 해결 |
|------|------|------|
| `search_entities` 결과 0건 | `p_entity_type: 'org'` 하드코딩 | `p_entity_type: null`로 변경 |
| `search_entities` 결과 0건 | `users` 테이블에 해당 guid 없음 | `users` 테이블에 먼저 INSERT |
| `PROFILE_NOT_FOUND` | `user_profiles`에 해당 guid/handle 없음 | SQL로 업체 재등록 |
| 주소 검색 안 됨 | `address` 컬럼이 NULL | `address` 도로명주소 UPDATE |
| 메뉴 표시 안 됨 | `extra.menu` 배열 없음 | `extra` 컬럼 UPDATE |
| INSERT 시 `ON CONFLICT` 오류 | `primary_guid`에 unique constraint 없음 | SELECT 후 INSERT 또는 UPDATE 분기 |
